AI 陪练系统技术设计

租房经纪人训练场景，基于留资窗口数据分析，支持持续对聊、实时复盘、窗口识别与红黄线诊断。

01 / Product Scope

从“话术闯关”升级为“数据驱动的留资窗口陪练系统”

系统不是替经纪人自动代聊，而是把历史会话里的留资窗口、有效理由、最佳轮次和风险信号沉淀成训练引擎，帮助新人学会“什么时候该推进、为什么能推进、怎么说不打扰”。

窗口识别

用用户意向、需求明确、留资理由、时机轮次和负向风险，判断当前是否适合索要联系方式。

持续陪练

客户画像、偏好、顾虑、历史表达保持一致，留资后继续推进看房、复访和防爽约。

实时复盘

经纪人点击后，系统输出客户状态、窗口分、红黄线、问题定位和下一句建议。

数据闭环

从 30w 历史会话提炼典型场景、有效理由、轮次窗口、失败话术和质检规则。

02 / Data Insights

核心数据结论：留资成功依赖“理由 + 时机 + 风险抑制”

数据分析不只是用来做报表，而是直接转化为训练系统里的窗口分、Agent 状态、提示策略和复盘口径。

最强理由：看房约看

用户表达看房、约看或询问看房时间，经纪人索要联系方式用于约看，留资率约 41.1%。

稳定理由：后续跟进/继续推荐

有效沟通后用于后续跟进约 31.51%；用户提出找房需求后用于推荐更多匹配房源约 31.88%。

辅助理由：微信沟通/房源确认

明确提到微信沟通约 29.46%；确认房源状态、价格、位置、入住时间后进一步确认约 33.33%。

分析结论 产品/技术转化 训练价值
1-3 轮索要电话留资率较低，从第 4 轮开始相对稳定 时机轮次分：1-3 轮谨慎，4-10 轮重点捕捉 训练新人避免开场强推电话
10 轮内结束比例更高，不及时把握会流失 实时复盘提示“当前强窗口”并给下一句 训练新人在窗口出现时及时推进
2 分钟到 12 分半区间留资率相对较高 会话时长作为 timing_score 特征 训练节奏感，而不是无限闲聊
明确拒绝、已租到、刚索要未回应时继续提示会造成打扰 负向风险分最高可扣 100，触发后暂停提示 训练经纪人识别停止推进和降噪时机

03 / 总体架构

总体架构：从真实数据到陪练、复盘和评分的完整链路

整体可以理解为五层：先把真实会话和用户行为加工成知识与特征，再由会话编排层驱动客户模拟、实时复盘和阶段评分，最后在经纪人工作台和教练后台里呈现。

数据输入层
历史会话客户消息、经纪人回复、轮次、时间戳、是否留资、是否约看。
用户行为浏览时长、收藏、查看地图、VR 浏览、房源卡点击。
房源信息价格、位置、户型、入住时间、看房状态、通勤信息。
质检规则隐私合规、虚假承诺、骚扰风险、平台禁止话术。
清洗脱敏、角色识别、会话切片、行为对齐、结果回标
知识与特征层
场景库预算敏感、强防备、急租、通勤顾虑、砍价、沉默流失。
客户画像库意向强弱、隐私顾虑、价格敏感、需求清晰度、回复风格。
话术与案例库成功留资路径、有效回复、失败话术、可复用表达。
窗口特征库强意向轮次、首次索要轮次、留资理由、会话时长、负向风险。
读取场景配置、难度参数、客户画像、房源上下文和窗口特征
会话编排层
会话运行器保存当前对话、客户记忆、训练目标、阶段状态和历史动作。
任务路由器根据用户动作分发到客户模拟、实时复盘、阶段评分或红黄线检查。
安全与规则拦截检查隐私诱导、虚假承诺、重复打扰和房源真实性风险。
根据“继续对聊 / 点击复盘 / 阶段结束”触发不同 Agent
Agent 能力层
场景导演生成本局客户、隐藏顾虑、训练目标、失败条件和难度参数。
客户模拟扮演客户持续对聊，保持画像稳定，不输出教练建议。
实时复盘包含状态跟踪子 Agent 和留资窗口评分子 Agent，输出诊断和建议。
评分教练阶段结束后输出能力评分、问题片段、训练建议和下一关推荐。
输出可解释结果：客户状态、窗口分、红黄线、下一句建议、阶段评分
应用输出层
经纪人陪练工作台场景选择、难度选择、持续聊天、快捷话术、房源卡。
AI 实时复盘面板当前诊断、留资窗口分、红黄线、下一步建议、阶段评分。
教练与管理后台场景配置、规则配置、学员报告、团队能力看板、效果追踪。

04 / Data Pipeline

数据加工链路：把会话、行为轨迹和留资结果转成窗口特征

1 清洗脱敏 去除手机号、姓名、地址、微信号，统一客户/经纪人角色和时间戳。
2 多源对齐 对齐浏览时长、收藏、地图、VR、房源卡曝光和聊天消息。
3 会话切片 按业务阶段切分：开场、需求、异议、留资、约看、复访。
4 标签标注 标注客户意图、强意向轮次、需求字段、留资理由和负向风险。
5 窗口挖掘 提取首次索要轮次、最近强意向轮次、会话时长和最终留资结果。
6 资产入库 形成场景库、画像库、话术库、窗口规则库、红黄线库和评测集。

05 / Agent Orchestration

Agent 分工要清楚：客户扮演和复盘评估必须隔离

Agent 输入 输出 关键约束
场景导演 Agent 场景、难度、训练目标、历史案例 客户画像、隐藏任务、异议脚本、失败条件 保证每局可控、可复现、可评分
客户模拟 Agent 画像、状态、最近对话、房源信息 客户下一句回复 不能泄露隐藏状态，不能主动教经纪人

06 / Conversation State

会话状态机：留资只是阶段节点，不是对话终点

1 首次咨询 目标：回应问题，建立基本服务感。
2 需求挖掘 目标：预算、区域、通勤、入住时间、人数。
3 异议处理 目标：承接价格、距离、房况、隐私顾虑。
4 留资 目标：说明用途，获得手机号或微信。
5 约看 目标：确认时间、地点、备选方案。
6 复访推进 目标：防爽约、看后跟进、二次推荐。

隐藏状态

  • trust_score：信任度
  • intent_score：租房意向
  • need_completeness：需求完整度
  • lead_window_score：留资窗口分
  • lead_reason_type：当前留资理由
  • last_strong_intent_turn：最近强意向轮次
  • annoyance_score：烦躁度
  • privacy_concern：隐私顾虑
  • lead_captured：是否已留资

07 / Difficulty Control

难度不是一句“客户更难搞”，而是一组可调参数

L1
初级

需求明确、回复积极、1 个主要异议、容错高。训练标准流程。

L2
中级

需求部分模糊、2-3 个异议、需要合理解释留资用途。训练节奏和承接。

L3
高级

客户防备强、回复短、需求摇摆、过早推进会失败。训练复杂判断。

参数 控制方式 对客户模拟的影响
初始意向 intent_score 初值 决定客户是否主动补充需求、是否愿意继续聊
隐私防备 privacy_concern 初值和触发阈值 决定客户是否抗拒留电话，以及需要怎样的解释
异议密度 objection_count 和 objection_script 控制价格、距离、押付、真实性等问题出现频率
容错率 annoyance_score 增长系数 控制经纪人答非所问、强推后客户是否流失

08 / Real-time Review

AI 实时复盘：由状态跟踪和窗口评分两个子 Agent 驱动

触发方式

经纪人点击“AI 实时复盘”后，实时复盘 Agent 先调用状态跟踪子 Agent 和留资窗口评分子 Agent，再读取当前会话、隐藏状态、规则命中和成功路径，生成当下诊断。

输出结构

  • 当前客户状态
  • 留资窗口分和触发证据
  • 经纪人已经完成的动作
  • 缺失动作和不合理回复
  • 红黄线命中说明
  • 下一步 2-3 条可发送话术

09 / Lead Window Score

留资窗口评分引擎：把数据结论落成可计算的分层规则

将用户行为、会话意图、需求成熟度、核心留资理由和时机轮次合并成 0-100 分。分数不是替代经纪人判断，而是实时复盘里的触发依据和提示阈值。

子分项 权重 核心输入 评分逻辑
用户意向分 25% 浏览时长、收藏、查看地图/VR、问价格/位置/是否还在/入住时间/想看房 行为意向 + 会话意向双通道融合，近期强意向表达加权更高
需求明确分 20% 预算、区域、户型、入住时间、通勤、租期、人数、特殊要求 按字段完整度加权，越接近可推荐和可约看，分数越高
核心留资理由分 35% 约看、问看房时间、确认房源状态、到店/到现场、后续推荐、微信沟通 依据历史留资率给理由类型赋值，看房约看类最高，后续跟进和继续推荐次之
时机轮次分 20% 会话总轮次、首次索要手机号轮次、最近强意向轮次、会话时长 1-3 轮谨慎，4-10 轮重点捕捉；2-12.5 分钟窗口期加权
负向风险分 -100% 明确拒绝、已租到、长时间未回复、刚索要未回应、烦躁/投诉风险 命中后直接降权或暂停提示，避免重复打扰
留资理由类型 历史留资率 评分含义 推荐提示方式
看房、约看、询问看房时间 41.1% 最强核心理由，lead_reason_score 最高 说明用于约看确认，给两个时间选项
有效沟通后用于后续跟进 31.51% 稳定理由，需已完成基本需求确认 说明只同步本次房源和进展，降低打扰感
用户提出找房需求，用于推荐匹配房源 31.88% 适合预算、区域、通勤已明确的场景 承诺发送 2-3 套匹配房源或对比方案
确认房源状态、价格、位置、入住时间 33.33% 适合房源确认型对话，理由明确但需快速履约 说明用于房东确认、临时变动通知
双方明确提到加微信或微信沟通 29.46% 辅助理由，需结合需求或看房动作增强 尊重客户偏好，避免跨平台强诱导
lead_window_score = intent_score * 0.25 + need_clarity_score * 0.20 + lead_reason_score * 0.35 + timing_score * 0.20 - negative_risk_penalty score = clamp(lead_window_score, 0, 100)

窗口分层

  • 0-39 低意向，不建议索要手机号
  • 40-59 有留资意图，继续了解需求
  • 60-74 弱窗口，可以委婉试探
  • 75-89 强窗口，提示经纪人
  • 90-100 最佳窗口，强提示并给话术

10 / Guardrails

红黄线：把业务风险和培训问题拆开治理

红线 严重合规风险

  • 虚假承诺：保证砍价、保证成交、保证房源仍在
  • 隐私诱导：不说明用途就索要手机号或微信
  • 不实房源：明知信息不确定仍肯定化表达
  • 歧视、威胁、骚扰、泄露其他客户隐私
  • 绕过平台规则，引导私下交易

黄线 转化和体验问题

  • 客户问题未正面回答
  • 连续追问，像审问而不是服务
  • 过早要电话，未建立留资理由
  • 只堆房源卖点，未结合客户需求
  • 客户高意向时推进不及时
GuardrailCheck = RuleEngine + LLMClassifier RuleEngine: - 正则检测手机号、微信、绝对化承诺、敏感词 - 检查平台禁止行为 LLMClassifier: - 判断是否答非所问 - 判断是否过早推进 - 判断是否虚假/夸大表达 - 输出风险等级、证据句、修正建议

11 / Evaluation

评分体系：先校准，再评估，再看业务结果

评分不是单次打分，而是一套三层体系：规则分判断窗口是否成立，模型分判断对话是否自然且合规，结果分判断训练是否真的推动留资和约看。

规则分

基于留资窗口分、红黄线、需求完整度、轮次时机等可解释规则，回答“此刻能不能推”。

模型分

由分类器或大模型判断自然度、同理心、答非所问、是否过早推进，回答“话说得好不好”。

结果分

看是否真正进入留资、约看、复访、成交前推进，回答“有没有业务效果”。

总评分公式

final_score = 0.35 * rule_score + 0.25 * model_score + 0.20 * stage_progress_score + 0.20 * compliance_score
  • rule_score：需求、异议、留资窗口、时机轮次
  • model_score：表达自然度、承接质量、推进节奏
  • stage_progress_score：留资、约看、复访推进结果
  • compliance_score：红黄线、隐私边界、虚假承诺

分数解释

  • 0-59 训练动作不完整，建议重新演练
  • 60-74 有推进意识，但窗口判断或表达质量不足
  • 75-89 能抓窗口，整体可用，仍可优化细节
  • 90-100 窗口判断和话术都比较稳定
评分维度 核心依据 建议校准方式 观察指标
留资窗口 用户意向、需求明确、留资理由、时机轮次、负向风险 用历史留资率和专家标注校准权重与阈值 lead_window_score、触发准确率
话术质量 是否答非所问、是否自然、是否有同理心、是否推进过快 专家抽样复核模型分，形成评分 Rubric model_score、一致性得分
阶段推进 是否完成留资、约看、复访、后续跟进 按训练场景定义阶段目标，不同场景不同权重 留资率、约看率、复访率
合规表达 红线、黄线、隐私说明、房源真实性、平台规则 先规则后模型，红线一票否决，黄线扣分 风险命中率、违规率
离线评测

用标注集做留资窗口、红黄线、意图识别、回复质量的准确率和一致性评估。

线上评测

看强窗口提示采纳率、平均对话轮次、窗口命中后留资率、过早索要率。

业务评测

看新人通关率、培训周期、留资率、约看率、质检投诉率是否改善。

12 / Delivery Plan

工程落地建议：先做可控 MVP，再接入更完整的训练闭环

阶段 1：MVP

10 个高频场景、3 档难度、持续对聊、留资窗口分、实时复盘、基础红黄线。

阶段 2：数据增强

接入行为轨迹、强意向轮次、成功路径挖掘、案例检索和窗口分校准。

阶段 3：运营闭环

教练后台、团队能力看板、个性化训练路径、真实业务指标对照。

模块 MVP 技术选型 后续增强
客户模拟 LLM + 场景 Prompt + 状态 JSON RAG 检索真实客户片段，增强口吻和异议真实度
状态跟踪 结构化输出 + 窗口特征抽取 小模型分类器识别强意向、需求字段和负向风险
