# 新增房源关注功能与关注列表 — 技术文档

**版本日期**: 2026-06-03

## 1. 架构设计

本次需求在现有 MVVM 架构上叠加了一个轻量级的**本地状态管理层**，不涉及后端交互：

```
PropertyDetailView / PropertyDetailScreen (详情页)
        │
        ├──→ FavoriteManager (iOS) / FavoriteViewModel (Android)
        │         └── 本地持久化 (UserDefaults / SharedPreferences)
        │
        └──→ FavoriteListView / FavoriteListScreen (列表页)
```

无网络层、无 Repository 层，数据完全本地存储。

## 3. 关键实现逻辑

### iOS: UserDefaults 持久化

```swift
class FavoriteManager: ObservableObject {
    @Published var favoriteIDs: [String] {
        didSet { UserDefaults.standard.set(favoriteIDs, forKey: key) }
    }
    init() {
        favoriteIDs = UserDefaults.standard.stringArray(forKey: key) ?? []
    }
}
```

关键点：`@Published` 的 `didSet` 中同步写入 `UserDefaults`，实现"内存状态 + 持久化"的双向绑定。各页面独立实例化 `FavoriteManager`，通过 `UserDefaults` 隐式同步状态。

### Android: SharedPreferences + StateFlow

```kotlin
class FavoriteViewModel(application: Application) : AndroidViewModel(application) {
    private val _favoriteIds = MutableStateFlow<Set<String>>(emptySet())
    val favoriteIds: StateFlow<Set<String>> = _favoriteIds.asStateFlow()

    fun toggleFavorite(propertyId: String) {
        val current = _favoriteIds.value.toMutableSet()
        // add / remove
        _favoriteIds.value = current
        prefs.edit().putStringSet("favorite_ids", current).apply()
    }
}
```

关键点：`AndroidViewModel` 持有 `Application` 上下文，可访问 `SharedPreferences`；`StateFlow` 保证 Compose 的响应式重绘；`viewModel()` 默认在 NavHost 范围内共享同一实例。

### Android: TopAppBar actions 槽位

```kotlin
TopAppBar(
    actions = {
        IconButton(onClick = { viewModel.toggleFavorite(property.id) }) {
            Icon(
                imageVector = if (isFavorite) Icons.Default.Favorite else Icons.Default.FavoriteBorder,
                tint = if (isFavorite) MaterialTheme.colorScheme.error else MaterialTheme.colorScheme.onSurfaceVariant
            )
        }
    }
)
```

## 4. 接口定义

### iOS

```swift
// FavoriteManager.swift
class FavoriteManager: ObservableObject {
    var favoriteIDs: [String] { get set }
    func isFavorite(property: Property) -> Bool
    func toggleFavorite(_ property: Property)
    func favoriteProperties(from all: [Property]) -> [Property]
}

// FavoriteListView.swift
struct FavoriteListView: View { }

// PropertyDetailView 调用点
@StateObject private var favoriteManager = FavoriteManager()
Button(action: { favoriteManager.toggleFavorite(property) }) { ... }
```

### Android

```kotlin
// FavoriteViewModel.kt
class FavoriteViewModel(application: Application) : AndroidViewModel(application) {
    val favoriteIds: StateFlow<Set<String>>
    fun isFavorite(propertyId: String): Boolean
    fun toggleFavorite(propertyId: String)
}

// FavoriteListScreen.kt
@Composable
fun FavoriteListScreen(
    onBackClick: () -> Unit,
    onPropertyClick: (Property) -> Unit,
    viewModel: FavoriteViewModel = viewModel()
)

// PropertyDetailScreen.kt
@Composable
fun PropertyDetailScreen(
    property: Property,
    onBackClick: () -> Unit,
    viewModel: FavoriteViewModel = viewModel()
)
```

## 6. 兼容性说明

- **iOS**: 目标 iOS 16.6，`UserDefaults` 为系统基础 API，所有版本兼容
- **Android**: minSdk 26，`SharedPreferences` + `StateFlow` 为基础 API，所有版本兼容
- **持久化格式**: iOS 存储为 `String` 数组，`UUID.uuidString` 作为 key；Android 存储为 `StringSet`，`Property.id` 作为 key
