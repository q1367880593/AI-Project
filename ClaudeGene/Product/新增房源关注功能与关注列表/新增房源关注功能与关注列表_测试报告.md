## 6. 遗留风险

- **持久化数据格式未做迁移设计**：当前 iOS 使用 `String` 数组存储 UUID，Android 使用 `StringSet` 存储 id。若未来 Property 的 id 格式变更，需考虑历史数据兼容
- **无关注上限控制**：用户可无限关注，理论上无上限（受限于本地存储容量）
- **纯 UI 展示页面无单元测试覆盖**：FavoriteListView / FavoriteListScreen 为纯展示组件，建议通过 Compose UI Test / XCUITest 补充端到端验证
- **未验证大规模关注列表性能**：当前 mockData 仅 5 条，若关注列表达到数百条，LazyColumn/List 性能需重新评估
