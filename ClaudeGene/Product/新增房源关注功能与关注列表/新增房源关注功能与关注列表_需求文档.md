# 新增房源关注功能与关注列表

**版本日期**: 2026-06-03

## 1. 需求背景与目标

为提升用户留存与房源管理体验，需在房源详情页增加"关注/取消关注"功能，并新增独立的"我的关注"列表页，方便用户快速查看已关注的房源。

## 2. 功能范围

**包含**:
- 房源详情页新增关注按钮（heart 图标），点击切换关注状态
- 关注状态持久化存储（本地），应用重启后保留
- 新增"我的关注"列表页，展示所有已关注房源
- 首页增加"我的关注"入口导航
- 关注列表支持点击跳转房源详情

**不包含**:
- 关注数据云端同步
- 关注数量上限控制
- 关注状态推送通知

## 5. 技术方案要点

1. **状态持久化**：iOS 使用 `UserDefaults.stringArray`，Android 使用 `SharedPreferences.getStringSet`
2. **响应式状态**：iOS 用 `ObservableObject` + `@Published`，Android 用 `StateFlow` + `collectAsState`
3. **跨页面状态共享**：iOS 各页面独立创建 `FavoriteManager` 实例，通过 `UserDefaults` 同步；Android 通过 `viewModel()` 获取单例 `FavoriteViewModel`
4. **复用现有组件**：关注列表直接复用房源卡片样式，保持视觉一致
5. **无新增依赖**：不涉及第三方库引入

## 6. 边界条件与异常场景

- 关注列表为空时展示空状态（heart.slash / FavoriteBorder 图标 + 提示文字）
- 房源详情页传入的 property 可能已不在 mockData 中，但关注状态仍保留
- UserDefaults/SharedPreferences 被手动清除后，关注数据丢失
