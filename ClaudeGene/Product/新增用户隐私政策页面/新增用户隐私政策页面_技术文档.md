# 新增用户隐私政策页面 — 技术文档

**版本日期**: 2026-05-26

## 1. 架构设计

本次需求为**纯展示型页面**，不涉及数据层、网络层或状态管理。架构上属于最简单的 View-Only 层：

```
SettingsView/SettingsScreen (入口)
        │
        ├──→ PrivacyPolicyView / PrivacyPolicyScreen (详情)
        │         └── 静态文本 + 滚动容器
        │
        └──→ AboutView (已有，参考对比)
```

无 ViewModel、无 Repository、无网络依赖。

## 3. 关键实现逻辑

### iOS: ScrollView 嵌套 VStack

使用 `ScrollView` 作为根容器，内部 `VStack(alignment: .leading, spacing: 16)` 堆叠内容。每个章节由 `sectionTitle` 和 `sectionBody` 两个私有方法生成，保证样式一致。

```swift
ScrollView {
    VStack(alignment: .leading, spacing: 16) {
        Text("隐私政策").font(.title).fontWeight(.bold)
        // 7 个章节...
        Spacer(minLength: 40)
    }
    .padding()
}
```

### Android: verticalScroll + rememberScrollState

Column 使用 `Modifier.verticalScroll(rememberScrollState())` 实现滚动。`SectionTitle` 和 `SectionBody` 为私有 Composable，复用 MaterialTheme 字体层级。

```kotlin
Column(
    modifier = Modifier
        .fillMaxSize()
        .verticalScroll(rememberScrollState())
        .padding(16.dp)
) { /* 7 个章节 */ }
```

### Android SettingsScreen 重构

原 `SettingsScreen` 为占位页（仅一个 Center 的 Text），本次重写为功能完整的设置列表：
- 使用 `ListItem` + `HorizontalDivider` 构建列表
- `Modifier.clickable` 处理点击事件（需 import `androidx.compose.foundation.clickable`）
- 接收 `onPrivacyPolicyClick: () -> Unit` 回调，由父级 NavHost 控制导航

## 4. 接口定义

### iOS

```swift
// PrivacyPolicyView.swift
struct PrivacyPolicyView: View { }

// SettingsView.swift 调用点
NavigationLink(destination: PrivacyPolicyView()) {
    Text("隐私政策")
}
```

### Android

```kotlin
// PrivacyPolicyScreen.kt
@Composable
fun PrivacyPolicyScreen(onBackClick: () -> Unit)

// SettingsScreen.kt
@Composable
fun SettingsScreen(
    onBackClick: () -> Unit,
    onPrivacyPolicyClick: () -> Unit
)

// MainActivity.kt 路由配置
composable("privacy_policy") {
    PrivacyPolicyScreen(onBackClick = { navController.popBackStack() })
}
```
