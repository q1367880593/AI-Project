 ClaudeGene — iOS 多智能体自动化 Agent 系统

 Context

 在 /Users/xiaolongbao/Documents/ClaudeGene 目录下，从零构建一个本地 CLI
 多智能体系统，能够接收 iOS 需求（自然语言或 PRD
 文件），自动完成需求拆解、OC/SwiftUI 代码编写、集成测试、ADHOC
 打包发布的全流程。串行执行，基于 Claude API +

 ---
 核心设计

 1. CLI 入口 (main.py)

 python main.py --project /path/to/TestProject \
                --requirement "新增用户登录页面，支持手机号+验证码"

 python main.py --project /path/to/TestProject \
                --prd requirements.md
 参数：
 - --project：Xcode 项目根目录（必填）
 - --requirement：自然语言需求（与 --prd 二选一）
 - --prd：PRD Markdown 文件路径（与 --requirement 二选一）
 - --skip-release：只跑到测试，不打包（可选）
 - --dry-run：只输出计划，不执行代码写入（可选）

 2. Master Agent (agents/master_agent.py)

 - 系统提示：iOS 技术负责人角色，负责任务拆解和决策
 - 输入：原始需求 + 项目结构快照
 - 输出：结构化任务列表（JSON），写入 task_state.json
 - 工具：list_files、read_file（读取项目结构）
 - 决策：根据需求判断是否需要新建文件 or 修改已有文件，选择 OC 还是 SwiftUI

 任务列表格式：
 {
   "session_id": "uuid",
   "requirement": "...",
   "tasks": [
     {"id": 1, "type": "requirements", "description": "...", "status":
 "pending"},
     {"id": 2, "type": "code", "files": ["LoginVC.m", "LoginVC.h"], "status":
 "pending"},
     {"id": 3, "type": "test", "status": "pending"},
     {"id": 4, "type": "release", "status": "pending"}
   ]
 }

 3. Requirements Agent (agents/requirements_agent.py)

 - 系统提示：iOS 产品/技术分析师
 - 输入：原始需求 + Master 的任务描述
 - 输出：细化需求文档（验收标准、涉及文件、技术方案），写入 memory/
 - 工具：read_file（读取相关已有代码）

 4. Code Agent (agents/code_agent.py)

 - 系统提示：资深 iOS 工程师，熟悉 OC 和 SwiftUI
 - 输入：细化需求 + 项目现有代码上下文
 - 工具：read_file、write_file、list_files、search_files
 - 行为：
   - 先读取相关文件理解现有代码风格
   - 生成代码后调用 write_file 写入项目
   - 支持多轮工具调用（先读后写）
 - 输出：写入的文件列表 + 变更摘要

 5. Test Agent (agents/test_agent.py)

 - 系统提示：iOS 测试工程师
 - 输入：变更文件列表 + 验收标准
 - 工具：read_file、write_file、run_xcodebuild
 - 行为：
   a. 生成单元测试/UI 测试文件，写入 Tests 目录
   b. 调用 xcodebuild test 运行测试
   c. 解析测试结果，失败时反馈给 Master（当前版本：记录到 state，不自动重试）
 - 输出：测试结果（pass/fail + 详情）

 6. Release Agent (agents/release_agent.py)

 - 系统提示：iOS DevOps 工程师
 - 输入：项目路径 + scheme + 测试通过状态
 - 工具：run_xcodebuild、read_file（读取 Info.plist 获取版本号）
 - 行为：
   a. xcodebuild archive 生成 .xcarchive
   b. xcodebuild -exportArchive 用 ADHOC 配置导出 .ipa
   c. 输出 .ipa 路径和构建信息
 - 输出：.ipa 文件路径 + 版本信息

 ---
 上下文记忆机制

 task_state.json（运行时状态）

 每个 Agent 执行完毕后更新对应 task 的 status 和 output：
 {
   "session_id": "...",
   "project_path": "...",
   "requirement": "...",
   "tasks": [...],
   "agent_outputs": {
     "requirements": {"acceptance_criteria": [...], "files_to_modify": [...]},
     "code": {"written_files": [...], "summary": "..."},
     "test": {"result": "pass", "details": "..."},
     "release": {"ipa_path": "...", "version": "..."}
   }
 }

 context_manager.py

 - load_state() / save_state()：读写 task_state.json
 - get_agent_context(agent_type)：为每个 Agent 组装上下文（前序 Agent 的输出）
 - save_session_history()：每次运行结束后归档到 memory/sessions/

 ---
 工具实现

 file_tools.py

 - read_file(path) → str
 - write_file(path, content) → bool
 - list_files(dir, extensions) → list[str]（支持过滤 .m/.h/.swift）
 - search_files(pattern, dir) → list[{file, line, content}]

 xcode_tools.py

 - find_workspace_or_project(dir) → str（自动检测 .xcworkspace 或 .xcodeproj）
 - find_schemes(project_path) → list[str]
 - run_xcodebuild(args) → {returncode, stdout, stderr}
 - build_test_command(project, scheme, destination) → str
 - build_archive_command(project, scheme, archive_path) → str
 - build_export_command(archive_path, export_path, options_plist) → str
